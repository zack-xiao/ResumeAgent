### 2. 基于 LLM 的智能接口文档自动生成平台（效能工具）

**技术栈**：Golang, LangChain, OpenAI/DeepSeek API, Go AST, Swagger/OpenAPI 3.0

**项目背景**：
在微服务快速迭代中，手动维护 Swagger/YApi 文档往往滞后于代码，导致文档与代码不一致，极大增加了前后端联调成本。为解决此痛点，自主研发了基于大模型的文档自动生成平台。

**核心职责与技术难点**：

- **代码静态分析与提取**：利用 Go AST (go/ast) 库对仓库源码进行深度静态分析，自动提取 Handler 中的请求参数（Struct）、路由定义及注释信息，构建精准的上下文（Context）作为 LLM 输入。
- **Prompt Engineering（提示词工程）**：设计结构化提示词，引导 LLM 将代码逻辑精准转换为标准的 OpenAPI 3.0 (Swagger) JSON 格式，成功解决了复杂嵌套结构体和泛型解析的准确性问题。
- **自动化集成与增量更新**：开发 CLI 工具集成至 CI/CD 流水线，实现"代码提交即文档更新"。设计文件哈希比对策略，仅对变更代码进行增量生成，大幅降低 Token 消耗成本。

**项目成果**：
文档编写效率提升 100%（实现零手动维护），接口解析准确率达到 95%+。在团队内部推广使用，覆盖 50+ 核心接口，显著降低了前期沟通与联调成本。
